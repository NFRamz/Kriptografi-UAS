# DIGIPROOF DEVELOPMENT RULES

You are NOT allowed to create placeholder pages.

You are NOT allowed to create:

* Coming Soon pages
* Under Construction pages
* Placeholder components
* Empty dashboards
* TODO sections
* Mock pages without functionality

Every feature requested must be implemented completely.

If a feature cannot be completed, ask for clarification instead of generating placeholders.

---

# DEVELOPMENT PRINCIPLES

Focus on one module at a time.

Do not generate the entire application at once.

Each module must be:

* Functional
* Tested
* Connected to Supabase
* Responsive
* Production ready

Before moving to the next module:

* Finish current module completely.
* Verify all requirements are implemented.

---

# CODE QUALITY RULES

Use:

* React
* TypeScript
* Vite
* Tailwind CSS
* Shadcn UI
* Framer Motion
* Supabase

Requirements:

* Clean Architecture
* Reusable Components
* Proper Folder Structure
* Strict TypeScript
* Error Handling
* Loading States
* Empty States
* Form Validation

---

# UI RULES

Every page must contain:

* Real UI
* Real interactions
* Responsive design
* Proper UX

Never use dummy buttons.

Never create buttons that do nothing.

Never create fake charts.

Never create fake forms.

---

# SUPABASE RULES

Use:

* Supabase Auth
* Supabase Database
* Supabase Storage

Implement:

* RLS
* Policies
* Validation

Use real Supabase queries.

Do not use hardcoded mock data unless explicitly requested.

---

# IMPLEMENTATION STYLE

When implementing a module:

1. Analyze requirements.
2. Design database schema.
3. Design components.
4. Design user flow.
5. Implement functionality.
6. Connect Supabase.
7. Handle errors.
8. Add loading states.
9. Test edge cases.
10. Verify completeness.

Only then move forward.
